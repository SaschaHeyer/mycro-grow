🍄 mycro-grow

Grow math for gourmet & functional mushroom farms — substrate hydration, spawn ratio, biological-efficiency yield, batch economics, substrate recipes, and fruiting-condition data for 14 species. Zero dependencies, MIT licensed.

mycro-grow is the open-source engine behind the free calculators at usemycro.com. Everything here already runs client-side in those tools — this package makes the same tested math reusable in your own scripts, dashboards, and grow-ops automation. Small mushroom farms run these numbers by hand or in fragile spreadsheets; mycro-grow packages them as clean, tested functions so you stop re-deriving them.

Install

npm install mycro-grow

Or just clone it — it's a handful of dependency-free ES modules. Source: github.com/SaschaHeyer/mycro-grow.

Quick start

import { growPlan } from 'mycro-grow';

// Plan a 100-block oyster batch: 5 (dry) per block, 60% moisture,
// 20% spawn rate, 85% biological efficiency, with costs and a $8 sale price.
const plan = growPlan({
  dryPerBlock: 5, blocks: 100, moisture: 60, spawnRate: 20, be: 85,
  costSub: 0.50, costSpawn: 1.00, extraPerBlock: 0.50, salePrice: 8,
});

plan.perBlock.spawnRatio;   // "1 : 5.0"
plan.totals.yield;          // 425   (fresh, total batch)
plan.economics.profit;      // 2850
plan.economics.margin;      // 83.82 (%)

Units: the library is unit-agnostic but internally consistent — pass masses in one unit (kg or lb) and prices per that same unit (e.g. $/lb). Every returned mass is in that unit. The numbers above are the library's real output (verified by the test suite).

API reference

Import from the package root: import { growPlan, substrateRecipe, yieldFromBE, hydration, spawn, economics, PRESETS, SPECIES_BE, FRUITING_CONDITIONS } from 'mycro-grow';

FunctionSignature → returnsWhat it does
hydration (dryMass, moisturePct) → {water, hydrated, moisture} Water to add to hit a target moisture (clamped 40–75%) and the resulting hydrated mass.
spawn (hydratedMass, spawnRatePct) → {spawn, rate, ratio} Grain spawn to add at a given rate, plus the human-readable ratio "1 : N".
yieldFromBE (dryMass, bePct) → number Expected fresh yield from biological efficiency: dry × (BE/100).
economics ({totalDry, totalSpawn, totalYield, blocks, costSub, costSpawn, extraPerBlock, salePrice}) → {totalCost, costPerBlock, revenue, profit, margin, costPerUnitYield} Per-batch cost, revenue, profit and margin.
growPlan ({dryPerBlock, blocks, moisture?, spawnRate?, be?, …costs}) → {perBlock, totals, economics} The full batch plan — hydration + spawn + yield + economics, per block and totalled. Mirrors the hosted Grow Calculator.
substrateRecipe ({preset?, basePerBlock, blocks, moisture?, pSaw?, pSoy?, pBran?, pGyp?, straw?}) → {perBlock, totals, ingredients, warnings} Scale a substrate recipe to N blocks. Presets: masters, suppsaw, soyhull, straw. Mirrors the hosted Substrate Recipe Calculator.
SPECIES_BE object Typical biological-efficiency ranges for 9 common gourmet/functional species.
FRUITING_CONDITIONS array (14 species) Temperature, humidity, CO₂, fresh-air and light targets per species — browse it as a 14-species reference table or the data behind the Fruiting Conditions cheat-sheet.

CLI

The package ships a mycro-grow binary — handy for quick farm math without writing code:

npx mycro-grow plan --dry 5 --blocks 100 --moisture 60 --spawn 20 --be 85 \
  --cost-sub 0.50 --cost-spawn 1.00 --extra 0.50 --price 8

npx mycro-grow substrate --preset masters --base 5 --blocks 100
npx mycro-grow species            # biological-efficiency table
